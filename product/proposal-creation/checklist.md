# Proposal Creation — Quality Gate Checklist

> Definition of done for the `proposal-creation` skill. Every applicable item must be
> checked, or waived with a one-line recorded reason. Unchecked, unwaived items
> block delivery.

**Skill:** `proposal-creation` · **Category:** Product · **Version:** 1.0.0

## Discover

- [ ] Problem statement and success criteria are written down
- [ ] Stakeholders and their concerns are identified
- [ ] Constraints (time, budget, tech, compliance) are explicit
- [ ] Existing artifacts reviewed for reuse

## Specify

- [ ] Intended outcome captured as a reviewable spec
- [ ] Scope and non-goals stated
- [ ] Assumptions and open questions listed

## Design — domain coverage

- [ ] Addressed: executive summary and value proposition
- [ ] Addressed: scope, deliverables, and explicit exclusions
- [ ] Addressed: pricing/effort estimate and commercial terms
- [ ] Addressed: timeline, milestones, and assumptions
- [ ] Addressed: risk, dependencies, and acceptance terms

## Validate — anti-patterns avoided

- [ ] Confirmed avoided: over-promising scope to win, then failing delivery
- [ ] Confirmed avoided: burying assumptions and exclusions in fine print
- [ ] Confirmed avoided: pricing without a defensible estimate behind it
- [ ] Spec is testable — acceptance criteria are objective
- [ ] Reviewed by domain expert(s)
- [ ] Risks identified with mitigations

## Deliver — artifacts

- [ ] Produced: Proposal document ready for stakeholder/client review
- [ ] Produced: Scope statement with exclusions
- [ ] Produced: Estimate and timeline with assumptions
- [ ] Final artifact stored in the spec registry
- [ ] Owner and review cadence assigned
- [ ] Handoff / communication done

## Sign-off

- [ ] All gates above checked or waived (with reason recorded)
- [ ] Artifact peer-reviewed by at least one other person
- [ ] Decisions/trade-offs captured (ADR or decision log)
- [ ] Owner and review cadence assigned
- [ ] Linked from the project's spec index / `openspec/` registry
